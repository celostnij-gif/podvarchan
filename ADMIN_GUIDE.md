## 1. Вход в админ-панель

**URL:** `https://podvarchan.com/admin/login`

Войти можно с email и паролем, указанными при первом запуске (seed). Если вы OWNER — у вас есть доступ ко всем функциям.

**Роли пользователей:**
| Роль | Права |
|------|-------|
| **OWNER** | Полный доступ: управление пользователями, настройки сайта, всё |
| **ADMIN** | Контент, публикация, CRM, медиа, журнал аудита |
| **EDITOR** | Создание и редактирование контента, публикация после REVIEW |
| **VIEWER** | Только просмотр (заявки, контент) |

## 2. Дашборд (`/admin`)

Главная страница админ-панели. Отображает:

- **Статистика:** количество послуг, статей, FAQ, отзывов, заявок, медиа-файлов, пользователей
- **Последние заявки:** последние 5 заявок из формы контакта (имя, email, дата, статус)
- **Черновики:** список неопубликованных статей и услуг (ссылки на редактор)
- **SEO-проблемы:** записи без meta-description (ссылки на SEO-редактор)

Если D1 база недоступна — дашборд покажет заглушку с предупреждением.

## 3. Управление услугами (`/admin/services`)

### Список услуг
Таблица со всеми услугами: иконка, название (RU), slug, категория, приоритет, статус, дата изменения.

**Статусы услуг:** `DRAFT` → `REVIEW` → `SCHEDULED` → `PUBLISHED` → `ARCHIVED`

### Редактор услуги (`/admin/services/[id]` или `/admin/services/new`)

**Основные настройки:**
- `slugBase` — уникальный идентификатор (латиница, используется в URL)
- `icon` — эмодзи-иконка (например, ✨, 🫂, 🌌)
- `category` — категория для группировки
- `priority` — приоритет (1 — высокий, 3 — низкий)
- `featured` — показывать в рекомендуемых

**Переводы (RU/UK):**
- `title`, `slug`, `shortTitle` — названия
- `description` — краткое описание
- `heroTitle`, `heroSubtitle` — заголовок и подзаголовок на странице услуги
- `symptomsJson` — JSON массив симптомов (с иконками)
- `processJson` — JSON массив шагов процесса работы
- `benefitsJson` — JSON массив преимуществ
- `faqJson` — JSON массив вопросов/ответов
- `ctaText` — текст кнопки призыва к действию

**Что можно делать:**
- Сохранить черновик (DRAFT)
- Опубликовать (PUBLISHED) — проходит SEO-валидацию
- Запланировать публикацию (SCHEDULED) — выберите дату и время
- Отправить на ревью (REVIEW)
- Архивировать (ARCHIVED)
- Удалить (только ADMIN/OWNER)
- Предпросмотр черновика (открывает публичную страницу с __preview cookie)

## 4. Управление блогом (`/admin/blog`)

### Список статей
Таблица: название, категория, статус, время чтения, дата.

**Статусы статей:** `DRAFT` → `REVIEW` → `SCHEDULED` → `PUBLISHED` → `ARCHIVED`

### Редактор статьи (`/admin/blog/[id]` или `/admin/blog/new`)

**WYSIWYG редактор (TipTap):**
- Жирный, курсив, зачёркнутый
- Заголовки H2-H4
- Цитаты
- Маркированные и нумерованные списки
- Блок кода
- Ссылки
- Изображения (из медиа-библиотеки)
- Отмена/повтор

**Поля статьи:**
- `title`, `slug` (RU/UK)
- `excerpt` — краткое описание для списка
- `content` — полный текст (TipTap редактор)
- `category` — категория блога
- `readingMinutes` — время чтения
- `status` — статус публикации

**При публикации:**
- Содержимое TipTap конвертируется в HTML
- Проходит SEO-валидацию (заголовок, описание, slug)
- Проверка на YMYL-слова (вылечим, гарантируем, и т.д.)

## 10. SEO-менеджер (`/admin/seo`)

Управление SEO-метаданными для всех типов контента.

**Что можно редактировать:**
- Title (meta-title)
- Description (meta-description)
- Keywords
- Canonical path
- OG:Title, OG:Description, OG:Image
- Robots: index/follow

**SEO Score:** От 0 до 100. Зелёный (80+) / жёлтый (60-79) / оранжевый (40-59) / красный (<40)

**Критерии оценки:**
1. Title существует (+10)
2. Длина title 30-65 символов (+10)
3. Description существует (+10)
4. Длина description 120-160 символов (+10)
5. Canonical указан (+10)
6. OG:Image выбран (+10)
7. OG:Title указан (+10)
8. OG:Description указан (+10)
9. Robots: index+follow (+10)
10. Schema type выбран (+10)

**Дополнительно:** Google Snippet Preview — как будет выглядеть страница в поиске Google.

## 17. Ревизии и предпросмотр

### Ревизии (версии)
При каждой публикации или ручном сохранении создаётся версия (ревізія) контента. Можно вернуться к любой предыдущей версии.

**Поддерживается для:** услуг, статей, страниц, FAQ, отзывов.

**При восстановлении:** контент возвращается в состояние DRAFT (не публикуется автоматически).

### Предпросмотр черновиков
Кнопка "Предпросмотр" в редакторе открывает публичную страницу с _preview cookie.
Вы видите черновик так, как он будет выглядеть после публикации.

> **Важно:** Ссылка предпросмотра работает только если вы авторизованы в админ-панели.
> При переходе непрошедший аутентификацию пользователь будет перенаправлен на страницу входа.

## 18. Часто задаваемые вопросы

### Как изменить пароль?
Войдите в админ-панель → Пользователи → Найдите своего пользователя → Edit → введите новый пароль → Save.

### Почему не публикуется статья/услуга?
Публикация блокируется, если:
- Отсутствует заголовок
- Отсутствует meta-description
- Slug пустой
- Текст содержит PLACEHOLDER, null, undefined
- Текст содержит ключи перевода (common.*, nav.*)

### Как восстановить удалённый контент?
Контент удаляется физически из БД. Если вы только что удалили — проверьте журнал аудита (`/admin/audit`) для контекста. Ревізіи хранят копии данных до момента удаления.

### Как добавить нового администратора?
OWNER → Пользователи → Создать → Указать email, пароль, имя, роль ADMIN/EDITOR/VIEWER.

### Почему не приходит email из формы контакта?
Проверьте:
1. `RESEND_API_KEY` настроен в Cloudflare secrets
2. `CONTACT_EMAIL` указывает на правильный адрес
3. В dev-режиме (`localhost`) отправка не происходит — только в production

### Где хранятся изображения?
По умолчанию — в `public/uploads/`. Для production рекомендуется R2 bucket (настройка описана в DEPLOY_CHECKLIST.md).

### Как часто публикуются запланированные посты?
Cron Worker запускается каждые 5 минут. Посты со статусом SCHEDULED и scheduledAt <= now() публикуются автоматически.

### Как сделать бекап базы данных?
```bash
npx wrangler d1 export DB --remote --output=./backup-$(date +%Y-%m-%d).sql
```
