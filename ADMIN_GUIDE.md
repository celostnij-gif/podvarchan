## 1. Вхід в адмінку

1. Перейдіть на [podvarchan.com/admin/login](https://podvarchan.com/admin/login)
2. Введіть **Email** та **Пароль**, які ви отримали при створенні акаунту
3. Натисніть **Увійти**
4. Після входу відкриється **Dashboard** зі статистикою:
   - Кількість послуг
   - Кількість заявок
   - Останні публікації

**Якщо забули пароль:** зверніться до розробника для скидання.

## 2. Створення/редагування послуги

### Нова послуга
1. У лівому меню оберіть **Послуги** → **Створити**
2. Заповніть поля для **RU** та **UK** (перемикання через вкладки):
   - **Назва** — заголовок послуги
   - **Коротка назва** — для карток/меню
   - **Опис** — детальний опис (TipPad редактор)
   - **Слоган** — підзаголовок на hero-секції
   - **Заголовок hero** — великий заголовок на сторінці
   - **Текст CTA** — текст кнопки "Записатися"
3. Додатково:
   - **Симптоми** — список симптомів (JSON)
   - **FAQ** — часті питання (JSON)
   - **Категорія** — група послуги
   - **Іконка** — emoji або icon name
4. Натисніть **Зберегти** або **Опублікувати**

### Редагування
- Перейдіть в **Послуги** → список
- Клікніть на послугу
- Внесіть зміни
- Натисніть **Зберегти**

---

## 3. Написання посту в блог (TipTap)

1. **Блог** → **Пости** → **Створити**
2. Вкладки **RU** / **UK**:
   - **Заголовок** — назва статті
   - **Опис** — короткий опис для прев'ю/картки
   - **Контент** — TipTap візуальний редактор:
     - Форматування: жирний, курсив, заголовки H2/H3
     - Списки (нумеровані, марковані)
     - Зображення
     - Посилання
     - FAQ-блоки
3. **Категорія** — оберіть із списку
4. **Статус** — Чернетка або Опубліковано
5. **Дата публікації** — встановіть або залиште поточну
6. **Зберегти** → **Опублікувати**

### Категорії блогу
- **Блог** → **Категорії**
- Можна створити, редагувати, змінювати порядок
- Кожна категорія може бути прив'язана до послуги

## 4. Створення нової сторінки (Page Builder)

1. **Сторінки** → **Створити**
2. Виберіть **Тип сторінки**:
   - `CUSTOM` — довільна сторінка з власним URL
   - Статичні типи (HOME, ABOUT, METHOD...) — зарезервовані системою
3. Заповніть **RU** / **UK**:
   - **Назва** — заголовок сторінки
   - **URL** — slug (наприклад, `moi-poslugy`)
4. **Секції** — додавайте блоки контенту:
   - `hero` — великий заголовок
   - `text-block` — текстовий блок
   - `image-text` — зображення + текст
   - `stats` — статистика/цифри
   - `timeline` — хронологія
   - `gallery` — галерея зображень
   - `video-embed` — вбудоване відео
   - `services-grid` — сітка послуг
   - `faq-group-ref` — посилання на групу FAQ
   - `testimonials-ref` — відгуки
   - `cta` — заклик до дії
   - `contact-form` — форма контакту
5. **Drag-and-drop** — перетягуйте секції для зміни порядку
6. **Toggle** — вмикайте/вимикайте окремі секції
7. **Зберегти**

---

## 5. Додавання відгуку

1. **Відгуки** → **Створити**
2. Заповніть поля:
   - **Ім'я клієнта**
   - **Текст відгуку**
   - **Оцінка** (1-5 зірок)
   - **Мова** — RU або UK
   - **Статус** — Чернетка або Опубліковано
3. **Зберегти**

Відгуки відображаються на головній сторінці в блоці "Що говорять клієнти".

---

## 6. Додавання FAQ

1. **FAQ** → **Створити**
2. Заповніть RU / UK:
   - **Питання**
   - **Відповідь**
   - **Група** (HOME, GENERAL, SERVICE, CONTACTS)
3. **Сортування** — порядок відображення
4. **Зберегти**

FAQ відображаються на сторінці `/faq/` та в блоці на головній.

---

## 7. Перегляд заявок

1. **Заявки** — список всіх заявок з форми контакту
2. Фільтри:
   - **Статус** — Нова, В роботі, Завершена, Відхилена
   - **Дата** — за період
3. Клікніть на заявку → **Деталі**:
   - Ім'я, телефон, email, повідомлення
   - **Історія подій** — всі дії з заявкою
   - **Нотатки** — внутрішні коментарі
4. Зміна статусу:
   - Оберіть новий статус
   - Додайте коментар (опціонально)

## 8. Налаштування SEO

1. **SEO** → список всіх сторінок сайту з SEO-оцінкою (0-100)
2. Для кожної сторінки:
   - **Title** (30-60 символів)
   - **Meta description** (70-160 символів)
   - **H1** — заголовок першого рівня
   - **Canonical URL**
   - **hreflang** — RU/UK симетрія
   - **Robots** — index/follow
   - **OG tags** — для соціальних мереж
   - **JSON-LD schema** — структуровані дані
3. **Google Snippet Preview** — як виглядатиме в пошуку
4. **YMYL-перевірка** — для медичних сторінок:
   - Автор
   - Дата
   - References
5. **Bulk-редагування** — зміна SEO для групи сторінок
6. **CSV експорт** — вивантаження SEO-аудиту

## 9. Завантаження зображень

1. **Медіа** → **Завантажити**
2. Виберіть файл (jpg, png, webp, svg, gif)
3. Додатково:
   - **Alt текст** — для SEO та доступності
   - **Підпис** — відображається під зображенням
4. Після завантаження зображення доступне в **Медіатека**
5. Скопіюйте URL зображення для вставки в контент

**Рекомендації:**
- Формат: WebP (найкраще співвідношення якість/розмір)
- Розмір: не більше 500 KB
- Роздільна здатність: 1200px по ширині для hero, 800px для контенту

---

## 10. Додавання редиректу

1. **Редиректи** → **Створити**
2. Заповніть:
   - **Звідки** — старий URL (наприклад, `/old-page`)
   - **Куди** — новий URL (наприклад, `/new-page`)
   - **Тип** — 301 (постійний) або 302 (тимчасовий)
   - **Статус** — Увімкнено/Вимкнено
3. **Зберегти**

**Лічильник:** кожен редирект показує кількість переходів (hit count).

---

## 11. Зміна навігації

1. **Навігація** → дерево меню HEADER
2. **Додати пункт:**
   - **Назва**
   - **Посилання** (URL)
   - **Розташування** — HEADER
3. **Drag-and-drop** — зміна порядку
4. **Увімкнути/Вимкнути** — показувати пункт чи ні
5. **Зберегти**

**Примітка:** UK-переклади назв пунктів синхронізуються окремо.

### Налаштування сайту
1. **Налаштування** → контактні канали:
   - Email, Telegram, WhatsApp
   - Додавання/редагування контактів
2. **Ключ-значення** — інші налаштування сайту

---

## 12. Журнал аудиту

1. **Аудит** — таблиця всіх дій в адмін-панелі
2. Фільтри:
   - **Тип сутності** (service, blog_post, page...)
   - **Користувач**
   - **Дата**
3. Кожен запис містить:
   - Хто зробив
   - Що змінив
   - Коли
   - IP-адреса

**Ревізії контенту:**
- **Ревізії** — історія змін контенту
- Можна переглянути попередні версії
- Відновлення попередньої версії
